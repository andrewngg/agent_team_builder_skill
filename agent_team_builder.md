# 多Agent团队生成引导 Skill v1.0

> 本 Skill 用于引导用户（人类）从零开始设计一个多 Agent 协作项目。  
> 适用场景：软件开发、内容创作、数据分析、产品设计、市场运营等任何复杂任务。  
> 核心理念：人类定义目标与边界，Agent 自主执行、验证、修复并形成闭环。

---

## 🎯 Skill 目标

* 通过一系列结构化问题，帮助用户明确项目目标、工作流、角色划分。
* 自动生成每个 Sub Agent 的职责、技能、输入输出、通信方式和责任归属。
* 最终输出一份完整的 `agents_team_spec.md` 文件，用户可直接在 Codex 或其他平台上部署。

---

## 📌 使用说明（对 Agent 的指令）

当你被调用时，请严格按照以下步骤**逐一询问**用户。每完成一步，等待用户回答后再进入下一步。最终汇总所有信息，生成规范的团队规格说明书。

---

## 🧭 引导流程（分模块执行）

### 第一步：项目总体目标确认

**请向用户提出以下问题（一次性全部提出，等待回答）：**

1. 你的项目名称是什么？
2. 用一句话描述项目的最终产出（例如：“开发一个 AI 写作助手 App”、“运营一个科技类短视频账号”）。
3. 这个项目的主要阶段或工作流是什么？（例如：规划 → 执行 → 验证 → 优化；或：选题 → 内容创作 → 素材制作 → 发布 → 复盘）

**记录回答，并总结为一个“项目概述”段落。**

---

### 第二步：核心任务模块拆解

**询问用户：**
4. 请列出你项目中的主要任务模块（至少 3 个，最多 10 个），并为每个模块起一个简短的名字（如“需求管理”、“代码开发”、“测试验收”、“内容撰写”、“视觉设计”、“数据分析”等）。
5. 为每个任务模块描述其具体职责（例如：“负责收集用户需求并撰写产品需求文档”）。

**将这些模块整理成一个列表，每个模块包含“名称”和“职责”。**

---

### 第三步：Sub Agent 角色与技能定义

**将每个任务模块转化为一个 Sub Agent，然后询问用户：**
6. 对于每个 Sub Agent，你需要它掌握哪些专属技能或工具？（例如：“擅长 Python 编程”、“熟练使用 Midjourney”、“熟悉 Salesforce API”等）
7. 每个 Sub Agent 的主要输入是什么？（它需要从哪些上游 Agent 或外部获取信息？）
8. 每个 Sub Agent 的主要输出是什么？（它产出什么？交付给谁？）

**为每个 Agent 填写以下表格字段：**  
`名称 | 职责 | 技能 | 输入 | 输出`

---

### 第四步：通信与协作机制设计

**询问用户：**
9. 你的 Agent 之间是串行工作（A 完成 → B 开始）还是可以并行工作（A 和 B 同时进行）？请描述整体工作流顺序。
10. 你希望 Agent 之间如何通信？（例如：通过发送 JSON 消息，包含任务描述、文件引用、状态码等）
11. 是否需要建立“中心管理 Agent”或“路由 Agent”来协调任务分配？还是采用点对点通信？
12. 如果某个 Agent 的输出不符合预期，应该如何处理？（选项：A. 由该 Agent 自行重新生成；B. 由下游 Agent 反馈修改意见；C. 由人类介入）

**根据回答，设计清晰的通信协议（消息格式示例）和工作流拓扑图（文字描述）。**

---

### 第五步：责任归属与修复闭环

**询问用户：**
13. 对每个 Sub Agent，谁负责最终验收它的工作？（可以是另一个 Agent，也可以是人类）
14. 当验收不通过时，修复工作由谁执行？（通常由出问题的 Agent 自己修复，除非涉及多个 Agent 的协作）
15. 如何记录问题与修复历史？（建议每个 Agent 维护自己的工作日志）
16. 如果问题反复出现（例如同一问题被驳回 3 次），是否有升级机制？（例如：通知人类管理者，或触发备用方案）

**整理出“责任矩阵”和“修复流程”。**

---

### 第六步：身份注册与工作空间管理

**询问用户：**
17. 你希望如何管理每个 Agent 的会话 ID 和身份信息？（推荐使用 JSON 文件或表格，由主 Agent 或人类初始化时注册）
18. 是否需要为每个 Agent 分配一个长期的工作空间（文件夹/数据库）来存储其产出和日志？

**给出一个“通讯录模板”示例（JSON 结构），并指导用户如何初始化。**

---

### 第七步：汇总与输出

**将所有信息整合成一份完整的 `agents_team_spec.md` 文件，必须包含以下章节：**

* 项目概述
* 全部 Sub Agent 列表（含名称、职责、技能、输入、输出）
* 工作流顺序与并行关系（文字描述或流程图）
* 通信协议与消息格式（含示例）
* 验收与修复责任矩阵（含升级策略）
* 初始化指南（如何创建各 Agent 会话并注册身份）
* 附录：给 Manager Agent 的初始启动 Prompt 模板

**最后，将这份 `agents_team_spec.md` 以代码块形式输出给用户，并提示用户保存并开始实施。**

---

## 📎 附录：通用模板与示例（仅供 Agent 参考）

### 示例：自媒体工厂项目（快速参考）

|Agent 名称|职责|技能|输入|输出|
|-|-|-|-|-|
|选题Agent|挖掘热点、制定选题列表|关键词检索、数据分析|用户兴趣标签|选题清单|
|内容Agent|撰写文章/脚本|写作、结构化表达|选题清单|初稿|
|素材Agent|生成配图、剪辑视频|图像生成、视频剪辑|内容初稿|视觉素材|
|复盘Agent|分析发布后数据，提出优化建议|数据分析、归因|发布后反馈数据|改进建议|

**工作流**：选题 → [内容 ∥ 素材] → 发布（人工/自动化，非 Agent 步骤）→ 复盘 → 反馈给选题。  
**通信格式**：`{"from": "选题Agent", "to": "内容Agent", "action": "新选题", "content": {...}}`  
**验收**：由复盘Agent验证内容效果，不通过则内容Agent修改。  
**修复闭环**：内容Agent自行修复，最多 3 次，否则升级到人类。

### 给 Manager Agent 的初始启动 Prompt 模板（可直接复制）

你好，你将作为本项目的【总管理Agent】。请严格按照以下规则执行：

1.【注册团队】：请立即创建一个 JSON 通讯录，包含所有 Sub Agent（见附件表格）。为每个 Agent 分配好职责边界和会话 ID。

2.【派发初始任务】：根据项目目标（见项目概述），向 {执行Agent} 发送第一条任务消息，并告知验收标准（依据 {验收Agent} 的规则）。

3.【启动闭环机制】：告知所有 Agent，当 {执行Agent} 完成任务后，必须自动通知 {验收Agent} 进行验收；验收不通过，{验收Agent} 必须带着修改意见直接回复给 {执行Agent}，{执行Agent} 收到后自行修正，直至 {验收Agent} 确认通过。

4.【日志记录】：要求每个 Agent 在执行每一步操作时，记录决策和产出到自己的工作日志。

5.【异常报告】：如果同一个问题被驳回超过 3 次，请直接向我发送警报摘要。

开始工作吧！

## ✅ Skill 结束语

当输出完 `agents_team_spec.md` 后，提示用户：

- 将此文件作为项目规划蓝图。

- 在 Codex 中创建对应的 Agent 会话，并使用上述初始化 Prompt 启动 Manager Agent。

- 如有调整，可随时重新运行本 Skill 进行迭代。
